12. Casos de uso principais
Crie uma seção detalhada para os 5 casos de uso principais.

Importante:
Não incluir login e cadastro como casos de uso principais. Eles devem aparecer apenas como requisitos obrigatórios de acesso simulado.

Caso de Uso 1 — Criar tarefa

Descrição:
Permite que o usuário cadastre uma nova tarefa no sistema.

Ator principal:
Usuário comum.

Pré-condição:
O usuário deve estar na área principal da aplicação.

Fluxo principal:
1. O usuário acessa o dashboard.
2. O usuário seleciona a opção de criar nova tarefa.
3. O sistema exibe o formulário de criação.
4. O usuário informa título, descrição, prioridade, status e prazo.
5. O usuário confirma o cadastro.
6. O sistema adiciona a tarefa à lista de tarefas.

Pós-condição:
A nova tarefa passa a ser exibida no dashboard.

Caso de Uso 2 — Listar tarefas

Descrição:
Permite que o usuário visualize as tarefas cadastradas.

Ator principal:
Usuário comum.

Pré-condição:
Existirem tarefas mockadas ou cadastradas durante a sessão.

Fluxo principal:
1. O usuário acessa o dashboard.
2. O sistema carrega a lista de tarefas.
3. O sistema exibe as tarefas em cards.
4. Cada card apresenta título, descrição resumida, prioridade, status e prazo.

Pós-condição:
O usuário visualiza suas tarefas organizadas.

Caso de Uso 3 — Visualizar detalhes da tarefa

Descrição:
Permite que o usuário acesse as informações completas de uma tarefa.

Ator principal:
Usuário comum.

Pré-condição:
A tarefa deve estar disponível na lista.

Fluxo principal:
1. O usuário acessa o dashboard.
2. O usuário seleciona uma tarefa.
3. O sistema abre a tela de detalhes.
4. O sistema exibe título, descrição, prioridade, status e prazo.

Pós-condição:
O usuário compreende melhor as informações da tarefa selecionada.

Caso de Uso 4 — Atualizar status da tarefa

Descrição:
Permite que o usuário altere o andamento de uma tarefa.

Ator principal:
Usuário comum.

Pré-condição:
A tarefa deve estar cadastrada no sistema.

Fluxo principal:
1. O usuário acessa os detalhes da tarefa.
2. O sistema exibe as opções de status.
3. O usuário escolhe entre Pendente, Em andamento ou Concluída.
4. O sistema atualiza o status da tarefa.
5. O dashboard reflete a alteração nos cards de resumo.

Pós-condição:
A tarefa passa a ter o novo status selecionado.

Caso de Uso 5 — Filtrar tarefas por status ou prioridade

Descrição:
Permite que o usuário filtre a visualização das tarefas.

Ator principal:
Usuário comum.

Pré-condição:
Existirem tarefas cadastradas ou mockadas.

Fluxo principal:
1. O usuário acessa o dashboard.
2. O sistema exibe opções de filtro.
3. O usuário seleciona um filtro, como Todas, Pendentes, Em andamento, Concluídas ou Alta prioridade.
4. O sistema atualiza a lista exibida conforme o filtro selecionado.

Pós-condição:
O usuário visualiza apenas as tarefas correspondentes ao filtro escolhido.
